알고리즘의 분석과 설계
같은 문제도 다양한 알고리즘으로 풀 수 있어요. 어떤 알고리즘이 더 효율적인지 비교·분석하고 직접 설계해봐요.
1 다양한 알고리즘을 비교·분석할 수 있다
2 문제에 적합한 알고리즘을 설계할 수 있다
🚑
1. 응급차 이송 알고리즘 비교

같은 목적(병원 도착)이라도 기준에 따라 경로가 달라져요. 두 알고리즘을 직접 비교해보세요!

🗺️ 집 → 병원 경로 지도 (숫자: 거리m / 분)

🔵 알고리즘 A 최단 거리
❶ 집에서 출발한다.
❷ 현재 위치에서 가장 짧은 거리로 이동하고, 한 번 이동한 지역은 다시 이동하지 않는다.
❸ 병원에 도착했다면 종료하고 그렇지 않으면 ❷를 수행한다.
📍 집 → 공원(100m) → 학교(400m) → 병원(300m)
총 거리: 1,100m / 총 시간: 18분
🔴 알고리즘 B 최단 시간
❶ 집에서 출발한다.
❷ 현재 위치에서 이동 시간이 가장 짧은 길로 이동하고, 한 번 이동한 지역은 다시 이동하지 않는다.
❸ 병원에 도착했다면 종료하고 그렇지 않으면 ❷를 수행한다.
📍 집 → 박물관(4분) → 학교(4분) → 병원(4분)
총 거리: 1,200m / 총 시간: 14분
🚑 결론: 응급차는 목숨이 걸린 상황이므로 거리보다 시간이 중요해요. 알고리즘 B(최단 시간)가 이 목적에 더 적합합니다!
❓ 응급차 이송 상황에서 A·B 중 더 적합한 알고리즘은 무엇이고, 그 이유는?

2. 알고리즘 설계 전략 4가지

카드를 클릭하면 자세한 예시를 볼 수 있어요!

🏆
가장 최선인 방법 선택하기
(탐욕 알고리즘)
매 순간 현재 상황에서 가장 최선인 방법을 선택
예) 응급차 알고리즘 B — 매 위치에서 가장 빠른 길을 즉시 선택해요. 자판기 거스름돈 계산도 매번 가장 큰 단위 동전부터 선택합니다.
✂️
문제 나누기
(분할 정복)
큰 문제를 작은 문제로 나눠 각각 해결
예) 전화번호부 검색 — 책의 절반을 펼쳐 앞/뒤를 판단하고 다시 절반씩 나눕니다. (이진 탐색)
🎲
시행착오 반복하기
(브루트 포스)
문제가 해결될 때까지 시행착오를 반복
예) 자물쇠 비밀번호 — 0000~9999 모든 경우를 하나씩 시도해서 맞는 번호를 찾습니다.
↩️
되추적하기
(백트래킹)
막히면 이전 단계로 돌아가 다른 방법 시도
예) 미로 찾기 — 막다른 길을 만나면 왔던 길로 돌아가 다른 갈림길을 선택합니다.
❓ "자물쇠 비밀번호 0000~9999를 모두 시도해본다"는 어떤 설계 전략인가요?
❓ 미로에서 막다른 길을 만났을 때 왔던 길로 돌아가 다른 길을 선택하는 전략은?

3. 나의 성취 수준 확인
⬜ 문제의 상태를 정의하고 수행 가능한 형태로 구조화할 수 있다.
⬜ 문제 해결을 위한 추상화의 중요성을 이해하고, 핵심 요소를 중심으로 알고리즘을 표현할 수 있다.
⬜ 알고리즘의 중요성을 이해하고, 문제를 해결하는 다양한 알고리즘을 비교·분석할 수 있다.
⬜ 사례를 중심으로 문제 해결에 적합한 전략을 선택하여 알고리즘을 설계할 수 있다.
